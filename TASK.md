# Техническое задание

## Задача
Нужна автоматизация для поиска квартир в районе Лилля (Франция). Скрипт выполняет поиск по заданным критериям, отправляет новые результаты в Telegram (кликабельные ссылки), а также ведёт два CSV‑файла:
1) **`daily_results.csv`** — все объявления, актуальные на дату запуска.
2) **`history_results.csv`** — накопительный файл со всеми актуальными ссылками; при каждом запуске добавляет новые объявления, проверяет актуальность старых и удаляет неактуальные (внутри очередного файла).

## Функциональные требования

### Поиск
- Источник: Google Custom Search API **или** прямые сайты‑доноры (список уточняется).
- Фильтры: Лилль и пригороды; тип жилья «квартира»; цена/площадь/комнаты — конфигурируемые параметры.

### Telegram
- Отправлять только новые объявления, без дублей.
- Формат: источник + заголовок + кликабельная ссылка.

### `daily_results.csv`
- Полный срез актуальных объявлений на дату запуска (например, 25 января 2026).
- «Актуальное» = страница доступна (HTTP 200) и нет признаков снятия (“annonce supprimée”, “annonce expirée”, “n’est plus disponible”, “vendu”).
- Файл **не пересоздаётся**, а **создаётся новый при каждом запуске** с таймстампом.

### `history_results.csv`
- Накопительный список **только актуальных** ссылок на момент запуска.
- При каждом запуске:
  - добавлять новые объявления,
  - проверять актуальность сохранённых ссылок,
  - удалять неактуальные (внутри итогового файла).
- Файл **не пересоздаётся**, а **создаётся новый при каждом запуске** с таймстампом.

### Именование CSV файлов (важно)
- Все CSV‑файлы создаются **новыми** на каждый запуск.
- Формат имени: `{YYYY-MM-DD}_{HH-MM-SS}_daily_results.csv` и `{YYYY-MM-DD}_{HH-MM-SS}_history_results.csv`.
- Примеры:
  - `2026-01-25_14-07-32_daily_results.csv`
  - `2026-01-25_14-07-32_history_results.csv`

### Данные в CSV
- Минимально: `date_checked`, `source`, `city`, `title`, `price_eur`, `rooms`, `area_m2`, `url`.
- Если данных нет — поле пустое.

## Нефункциональные требования
- Конфигурация через переменные окружения (API‑ключи, Telegram токен, chat ID).
- Ограничение запросов (sleep/лимиты).
- Логирование: сколько найдено/добавлено/удалено/неактуально.

## Выходные артефакты
- CSV‑файлы с таймстампом при каждом запуске.
- Python‑скрипт(ы), запускаемые командой `python ...`.

## Критерии готовности
- Telegram получает только новые объявления.
- `daily_results.csv` содержит актуальный срез на дату запуска.
- `history_results.csv` содержит только актуальные ссылки, без неактуальных/битых.
- После каждого запуска создаются новые CSV‑файлы с таймстампом; предыдущие остаются.
